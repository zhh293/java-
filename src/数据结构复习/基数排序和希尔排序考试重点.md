# 基数排序和希尔排序考试重点

## 一、基数排序（Radix Sort）

### 1.1 算法原理
- **核心思想**：按位数切割数字，从最低位到最高位依次排序
- **实现方式**：对每一位使用计数排序
- **稳定性**：稳定排序算法

### 1.2 排序过程详解
**示例数组**：[170, 45, 75, 90, 2, 802, 24, 66]

#### 第1轮（个位排序）：
- 提取个位数字：[0, 5, 5, 0, 2, 2, 4, 6]
- 按个位排序后：[170, 90, 2, 802, 24, 45, 75, 66]

#### 第2轮（十位排序）：
- 提取十位数字：[7, 9, 0, 0, 2, 4, 7, 6]
- 按十位排序后：[2, 802, 24, 45, 66, 170, 75, 90]

#### 第3轮（百位排序）：
- 提取百位数字：[0, 8, 0, 0, 0, 1, 0, 0]
- 按百位排序后：[2, 24, 45, 66, 75, 90, 170, 802]

### 1.3 考试题型示例
**题目**：对数组 [329, 457, 657, 839, 436, 720, 355] 进行基数排序，写出每轮排序结果。

**解答**：
- 初始：[329, 457, 657, 839, 436, 720, 355]
- 第1轮（个位）：[720, 355, 436, 457, 657, 329, 839]
- 第2轮（十位）：[720, 329, 436, 839, 355, 457, 657]
- 第3轮（百位）：[329, 355, 436, 457, 657, 720, 839]

### 1.4 算法特点
- 时间复杂度：O(d×(n+k))
- 空间复杂度：O(n+k)
- 稳定性：稳定
- 适用：整数或固定长度字符串

## 二、希尔排序（Shell Sort）

### 2.1 算法原理
- **核心思想**：缩小增量排序，分组进行插入排序
- **改进点**：对插入排序的优化
- **稳定性**：不稳定排序算法

### 2.2 排序过程详解
**示例数组**：[49, 38, 65, 97, 76, 13, 27, 49, 55, 4]

#### 增量序列：5, 3, 1

#### 第1轮（增量=5）：
- 分组：[49,13], [38,27], [65,49], [97,55], [76,4]
- 排序后：[13, 27, 49, 55, 4, 49, 38, 65, 97, 76]

#### 第2轮（增量=3）：
- 分组：[13,55,38], [27,4,65], [49,49,97], [55,76]
- 排序后：[13, 4, 49, 27, 38, 55, 49, 65, 97, 76]

#### 第3轮（增量=1）：
- 整体插入排序：[4, 13, 27, 38, 49, 49, 55, 65, 76, 97]

### 2.3 考试题型示例
**题目**：对数组 [81, 94, 11, 96, 12, 35, 17, 95, 28, 58, 41, 75, 15] 进行希尔排序，使用增量序列 5, 2, 1，写出每轮排序结果。

**解答**：
- 初始：[81, 94, 11, 96, 12, 35, 17, 95, 28, 58, 41, 75, 15]
- 第1轮（增量=5）：
  - 分组：[81,35,41], [94,17,75], [11,95,15], [96,28], [12,58]
  - 排序后：[35, 17, 11, 28, 12, 41, 75, 15, 96, 58, 81, 94, 95]
- 第2轮（增量=2）：
  - 分组：[35,11,12,15,81,95], [17,28,41,96,58,94]
  - 排序后：[11, 17, 12, 28, 15, 41, 35, 58, 81, 94, 96, 95, 75]
- 第3轮（增量=1）：
  - 整体插入排序：[11, 12, 15, 17, 28, 35, 41, 58, 75, 81, 94, 95, 96]

### 2.4 增量序列
- **Shell增量序列**：n/2, n/4, n/8, ..., 1
- **Knuth增量序列**：(3^k-1)/2
- **Sedgewick增量序列**：更复杂的数学序列

### 2.5 算法特点
- 时间复杂度：O(n^1.3) 到 O(n^2)（取决于增量序列）
- 空间复杂度：O(1)
- 稳定性：不稳定
- 适用：中等规模数据排序

## 三、考试重点对比

| 特征 | 基数排序 | 希尔排序 |
|------|----------|----------|
| 排序类型 | 分配式排序 | 插入排序改进版 |
| 时间复杂度 | O(d×(n+k)) | O(n^1.3~n^2) |
| 空间复杂度 | O(n+k) | O(1) |
| 稳定性 | 稳定 | 不稳定 |
| 适用场景 | 整数/字符串排序 | 一般数值排序 |
| 排序方向 | 从低位到高位 | 按增量分组 |

## 四、解题技巧

### 4.1 基数排序解题步骤
1. 确定最大数的位数
2. 从个位开始逐位排序
3. 每位使用计数排序
4. 保持稳定性（相等元素相对位置不变）

### 4.2 希尔排序解题步骤
1. 确定增量序列
2. 按增量分组
3. 对每组进行插入排序
4. 减小增量，重复步骤2-3
5. 最后增量为1时整体插入排序

## 五、常见错误
1. **基数排序**：忘记从最低位开始排序
2. **希尔排序**：增量序列计算错误
3. **稳定性理解**：基数排序是稳定的，希尔排序是不稳定的
4. **分组错误**：希尔排序分组时索引计算错误

## 六、考试应试策略
1. **基数排序**：重点掌握按位排序的过程，注意稳定性的体现
2. **希尔排序**：重点掌握分组和插入排序的结合，注意增量序列的选取
3. **时间复杂度**：记住各自的时间复杂度公式
4. **稳定性判断**：能够准确判断两种排序的稳定性
